# Blog Writing Guidelines
## The Lightroom Studios Blog

This document outlines the complete process for creating high-quality, SEO-optimized blog posts for The Lightroom Studios Blog.

---

## Step 1: Research & Planning

### Topic Research
- Identify target audience pain points
- Research keyword opportunities (primary and secondary keywords)
- Analyze competitor content
- Determine content angle/unique perspective

### Content Structure Planning
- Define main topic and subtopics
- Create preliminary outline with H2 and H3 headings
- Identify key questions to answer
- Plan content length (aim for 2,000-4,000 words for comprehensive guides)

### Audience Intent
- Informational: Educational content, how-to guides
- Commercial: Product/service comparisons, recommendations
- Transactional: Service pages, contact-focused content

---

## Step 2: Write Core Content

### Opening Hook
- Start with relatable scenario or problem statement
- Use conversational, authentic voice
- Establish credibility early
- Promise value/solution

### Body Content
- Write naturally, focusing on value and clarity
- Use specific examples and concrete details
- Include practical, actionable advice
- Break up text with subheadings every 300-400 words
- Use bullet points and numbered lists for scannability

### Voice & Tone
- Professional but approachable
- Direct and honest (avoid marketing fluff)
- Conversational (use "you", "I", "we")
- Authoritative without being condescending

### Content Quality Checklist
- [ ] Answers the main question thoroughly
- [ ] Provides unique insights or perspective
- [ ] Includes practical examples
- [ ] Addresses common objections/concerns
- [ ] Natural flow between sections
- [ ] No jargon without explanation

---

## Step 3: Add Visual Elements

### Images
- Hero image (featured image)
- Supporting images throughout content
- Screenshots or diagrams where helpful
- Ensure all images are high-quality and relevant

### Other Visual Elements
- Blockquotes for important tips or statistics
- Code blocks (if applicable)
- Tables for comparisons
- Pull quotes to highlight key points

---

## Step 4: Create Table of Contents

- Add jumplinks for all H2 sections
- Place near top of article (after intro)
- Use descriptive anchor text
- Improves user experience and SEO

---

## Step 5: Write FAQ Section (if applicable)

- Address common questions not covered in main content
- Use schema markup for FAQ (add structured data)
- Keep answers concise but complete
- Based on actual user questions when possible

---

## Step 6: SEO Optimization & Enhancement

**THIS IS WHERE SEO WORK HAPPENS - AFTER CONTENT IS WRITTEN**

### Meta Information
- [ ] Page title (50-60 characters, include primary keyword)
- [ ] Meta description (150-160 characters, compelling and keyword-rich)
- [ ] URL slug (short, descriptive, includes primary keyword)
- [ ] Canonical URL
- [ ] Open Graph tags for social sharing
- [ ] Twitter Card tags
- [ ] Article schema markup (JSON-LD)

### Keyword Optimization
- [ ] Primary keyword in H1 (title)
- [ ] Primary keyword in first 100 words
- [ ] Secondary keywords in H2 headings (naturally)
- [ ] Keywords in image alt text
- [ ] LSI (Latent Semantic Indexing) keywords throughout
- [ ] Avoid keyword stuffing - maintain natural flow

### Internal Linking
- [ ] Add 3-5 internal links to related blog posts
- [ ] Link to relevant service pages
- [ ] Link to main website pages
- [ ] Use descriptive anchor text (not "click here")
- [ ] Related articles section at bottom

### External Linking
- [ ] Add 2-4 authoritative external links
- [ ] Link to industry sources, research, tools
- [ ] Use rel="noopener" for security
- [ ] Verify all external links work
- [ ] External links add credibility and context

### Image Optimization
- [ ] Descriptive, keyword-rich alt text for all images
- [ ] Compress images for fast loading
- [ ] Use lazy loading (loading="lazy") except hero image
- [ ] Include captions/descriptions where helpful
- [ ] **Image credits:** Extract author name from filename and include proper attribution (e.g., "Image by [Author] from [Source]")
  - Format: Check filename for author information (e.g., "Image_by_kconcha_from_Pixabay")
  - Add credit in caption using format: "Image by [Author] from [Source]"
  - Place after image caption text, separated by " | " if both present

### Content Enhancements
- [ ] Add breadcrumb navigation
- [ ] Ensure proper heading hierarchy (H1 → H2 → H3)
- [ ] Add reading time estimate
- [ ] Include author byline
- [ ] Add publish/update dates
- [ ] Add tags/categories

---

## Step 7: Add Calls-to-Action (CTAs)

**Strategic CTA Placement:**

### Early Article CTA (20-30% through content)
- Subtle, service-related
- Colored info box format
- Example: "Need professional services? Learn more →"

### Mid-Article CTA (50-60% through content)
- More direct
- Free consultation or quote offer
- Example: "Ready to get started? Request a quote →"

### Final CTA (Bottom of article, before footer)
- Primary conversion goal
- Prominent design (gradient background, multiple buttons)
- Multiple contact options
- Should be the strongest, most conversion-focused

### ~~Newsletter Signup~~
- **REMOVED** - Does not fit WSJ newspaper aesthetic
- Focus on direct CTAs for services instead

### Related Articles
- 3-4 related blog posts
- Increases time on site
- Improves internal linking
- Placed near bottom

### CTA Best Practices
- No more than 3 CTAs per article (early, mid, final)
- Match classic WSJ newspaper design (borders, no gradients)
- Space them out naturally in content
- Each CTA should feel relevant to surrounding content
- Use professional, understated language (not salesy)
- Provide clear value proposition
- NO modern web design elements (gradients, rounded corners, bright colors, emojis)
- Use classic typography and simple borders only

- [ ] Mobile-friendly formatting
- [ ] Proper line spacing and typography

### Content Review
- [ ] Read entire article aloud
- [ ] Check for grammatical errors
- [ ] Verify all facts and statistics
- [ ] Ensure logical flow
- [ ] Remove redundancy
- [ ] Check tone consistency

### Link Verification
- [ ] All internal links work
- [ ] All external links work
- [ ] Links open correctly (new tab for external)
- [ ] Anchor links jump to correct sections

### Visual Review
- [ ] All images display correctly
- [ ] Alt text present on all images
- [ ] Proper image sizing
- [ ] CTAs display properly
- [ ] Formatting looks good on mobile

- Share on social media

## Content Templates

### Standard Article Structure
```
1. Hero Image
2. Breadcrumb Navigation
3. Article Header (Category, Title, Meta, Tags)
4. Opening Hook (2-3 paragraphs)
5. Table of Contents
6. Main Content (with H2/H3 sections)
   - Early CTA (after first major section) - simple bordered box
   - Supporting images
   - Mid-Article CTA (middle of content) - simple bordered box
7. FAQ Section (if applicable)
8. Conclusion/Summary
9. Final CTA (double border box - traditional newspaper ad style)
10. Related Articles (newspaper grid with vertical dividers)
11. Back to Blog Link
```

**Design Philosophy:**
All elements must match the Wall Street Journal newspaper aesthetic:
- Classic serif and sans-serif typography
- Black borders and dividers
- Off-white newspaper background
- No modern web design elements
- Professional and understated

### CTA Template Styles (WSJ Newspaper Style)

**All CTAs should match the classic newspaper aesthetic:**
- Simple black borders (no gradients or modern colors)
- Off-white backgrounds matching the newspaper
- Classic typography (serif headlines, sans-serif labels)
- Understated and professional
- No rounded corners, no shadows, no emojis

## Keywords & SEO Strategy

### Primary Keyword
- Use in: Title (H1), first paragraph, URL, meta description, 1-2 H2s, image alt text
- Density: 1-2% of total content
- Natural placement only

### Secondary Keywords
- Use in: H2/H3 headings, throughout body content, image alt text
- Related terms and variations
- Support primary keyword

### Long-Tail Keywords
- Address specific questions/searches
- Lower competition
- Higher conversion potential
- Use in FAQ sections

## Writing Style Guide

### DO:
✓ Write in active voice
✓ Use short sentences (15-20 words average)
✓ Break up long paragraphs (3-4 sentences max)
✓ Use contractions (you're, we're, it's)
✓ Start sentences with conjunctions occasionally (But, And, So)
✓ Include personal experiences/examples
✓ Use specific numbers and data
✓ Address reader directly ("you")
✓ Match WSJ newspaper aesthetic in all design elements
✓ Use classic borders and dividers

### DON'T:
✗ Use passive voice excessively
✗ Include marketing fluff/buzzwords
✗ Write walls of text
✗ Use jargon without explanation
✗ Make unsupported claims
✗ Be overly formal or stiff
✗ Keyword stuff
✗ Write generic content
✗ Use gradients, rounded corners, or modern web design
✗ Use emojis in CTAs or promotional content
✗ Use bright colors or flashy design elements

## Quality Standards

### Minimum Requirements
- 1,500+ words for standard posts
- 2,500+ words for comprehensive guides
- 3-5 internal links
- 2-4 external authoritative links
- 2+ images (hero + supporting)
- Table of contents for 1,500+ word posts
- At least 2 CTAs
- Mobile responsive
- Fast page load (<3 seconds)

### Excellence Indicators
- Unique insights or perspective
- Original research or data
- Comprehensive coverage
- Clear, actionable advice
- Strong examples
- Professional visuals
- Strategic CTA placement
- Well-optimized for SEO

### Monthly

- Review analytics for top posts
- Update dates if content revised

### Quarterly
- Review and update statistics
- Add new sections if relevant
- Refresh meta descriptions
- Update screenshots/images

### Annually
- Comprehensive content audit
- Major updates for evergreen content
- Keyword research refresh
- Competitor analysis

---

## Notes

- SEO optimization should ENHANCE good content, not replace it
- Write for humans first, search engines second
- CTAs should feel natural, not intrusive
- Quality over quantity - one great post beats three mediocre ones
- Authenticity and expertise build trust
- Data and examples increase credibility
- Always provide value to the reader
